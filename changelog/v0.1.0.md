# BookUnbind v0.1.0 — Initial Release

First public release of **BookUnbind**, a macOS / Windows / Linux desktop tool that splits a PDF e-book into per-page PDFs while preserving *both* the original PDF page index (as a three-digit prefix) and the detected book page label (as a suffix).

Representative filenames:

| Source situation | Produced filename |
|---|---|
| PDF page 21, book page `20` | `021_Page_20.pdf` |
| PDF page 25, book page `1-4` (chapter-style) | `025_Page_1-4.pdf` |
| PDF page 410, appendix `D-11` | `410_Page_D-11.pdf` |
| PDF page 5, front matter `iii` | `005_Page_iii.pdf` |
| PDF page 1, cover (no detection) | `001.pdf` |

---

## ✨ Highlights

### Smart page-number detection

Powered by PyMuPDF text + bounding box extraction. Only the bottom 7% of each page is scanned, and matches are ranked by pattern priority + vertical position (bottom-most wins).

Recognised patterns (priority order):

1. **Chapter-style** — `1-4`, `12-3` (very common in Taiwanese tech books)
2. **Alphabetic chapter code** — `A-3`, `D-11`
3. **Chinese chapter code** — `附錄-5`
4. **Arabic page number** — `1`..`9999`
5. **Lowercase Roman numeral** — `iii`, `xii` (front matter)
6. **Three-level hierarchical** — `1.2.3`

False-positive filter: rejects the whole line if it contains `$`, `NT`, `US`, `ISBN`, `售價`, `定價`, `建議`, `Tel`, `電話`, `傳真`, `專線`, `©`. This avoids matching prices, phone numbers, barcodes, etc.

### Dry-run preview workflow

- Pick PDF → auto-filled output directory (`{pdf_dir}/{pdf_stem}_unbind`) → **preview** → inspect the full filename list → **split**
- No files are written during preview
- Progress bar + live status during actual split
- Overwrite-confirmation dialog when the target directory already contains files
- Completion dialog with "open output folder" shortcut

### Modern dark-mode GUI (Flet 0.84)

- Click-to-pick drop-zone card with clear visual state (grey → green check after selection)
- Auto-disabled Split button until preview is confirmed
- Tooltips showing full paths on hover
- All long-running work runs off the UI thread via `asyncio.to_thread`

### Three-platform CI release pipeline

On every `v*` tag push, GitHub Actions builds `.app` (macOS), `.exe` bundle (Windows) and native binary (Linux) in parallel, then publishes a GitHub Release with all three archives attached.

- macOS: ditto-archived `.app` bundle
- Windows: `Compress-Archive` zip of build output
- Linux: `tar.gz` of build output

### Tests

27 unit tests covering the filename builder and page detector, including:

- All 6 supported page-number patterns
- Rejection of ISBN classification numbers (`312.83`) and ambiguous uppercase Roman (`DLI`)
- Chapter-page preference over plain numbers
- Top-of-page text correctly ignored
- Long footer notes filtered out
- Chinese chapter codes via PyMuPDF's built-in CJK font

## ⚠️ Known limitations

- **Native OS file drag-and-drop is not supported.** Flet 0.84 has no file-drop API for the desktop runtime yet; the UI falls back to click-to-pick.
- **Scanned PDFs (no text layer)** fall back to prefix-only filenames for every page. OCR support is planned for a later release.
- **Encrypted PDFs** must be decrypted before processing.

## 🧪 Technical notes

- **Python** 3.10+ (CI uses 3.12)
- **Flet** 0.84 (Flutter-based GUI; Flutter SDK 3.41.4 pinned in CI)
- **PyMuPDF** ≥ 1.24 (text and coordinate extraction)
- **pypdf** ≥ 4.0 (per-page PDF writing)

### Notable workarounds baked in

- **macOS shutdown crash fix.** PyMuPDF's static C++ diagnostic callback segfaults during Python interpreter teardown. `page.on_close` now calls `os._exit(0)` to bypass the destructor path entirely.
- **Windows CI UTF-8 fix.** Flet CLI prints Unicode spinner characters (`●`) that crash under Windows' default `cp1252` codepage. The release workflow sets `PYTHONIOENCODING=utf-8` + `PYTHONUTF8=1`.
- **Flet `flet build` prompt bypass on CI.** Flet expects a specific Flutter version at `~/flutter/{version}/`; the workflow uses `subosito/flutter-action` to install Flutter 3.41.4 and symlinks it into place, so the interactive SDK-download prompt never fires.
